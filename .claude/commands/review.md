## Steps

1. Review the changes to full the rules in the CLAUDE.md.
2. If everything is good, Approve
3. If changes are required/recommended, categories the changes in Critical, Warnings. no Nits!
4. Drop one very short comment on the PR page mentioning the issues.

## Guides

- Keep the communication to the user as short as possible with less reads
- The proper approach is not between options. It is always one single approach.
- Existing code is not 100% correct. So we are open for new approaches
- Analyse all the files of the PR without skipping any of them
- Do not skip any files subject to review
- Work autonomously without any questions and finish the review
