## Steps

1. Plan the requested task by considering the rules in the CLAUDE.md
2. Ask question if needed to finalize the plan only using the question tool
3. Checkout to a branch and start to Implement the plan by sticking to the rules of the CLAUDE.md.
4. Have a round of review and fix before reporting
4. When finished, commit and push and then open a PR.
5. Share the PR link so user can verify.

## Guides

- Keep the communication to the user as short as possible with less reads
- Use question tool if you need to ask questions
- The proper approach is not between options. It is always one single approach.
- Existing code is not 100% correct. So we are open for new approaches

## Important

- Run output through an opus agent
- Make sure you have a complete picture of the implementation on the plan mode to not stop and change the approach or hit blockers.
