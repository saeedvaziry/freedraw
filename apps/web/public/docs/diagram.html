Diagram code

Turn text into a diagram and back. FreeDraw understands a focused subset of Mermaid flowchart syntax — write nodes and edges as text, click Generate diagram, and they appear on the canvas. The panel also shows your current canvas as code, so you can copy it back out.

Quick start

flowchart TD
A[Start] --> B{Decision}
B -->|yes| C[Done]
B -->|no| A

The first line sets the diagram and its direction. Each following line declares a node or connects nodes with an edge.

Direction

Start with flowchart or graph, followed by a direction. The default is TD.

KeywordMeaning
TD / TBTop to bottom
BTBottom to top
LRLeft to right
RLRight to left

Nodes

A node is an id followed by a label wrapped in brackets — the bracket style picks the shape. The id is used to reference the node in edges; the label is what shows on the canvas.

SyntaxShape
A[Text]Rectangle
A(Text)Rounded rectangle
A((Text))Ellipse
A{Text}Diamond
A{{Text}}Hexagon
A[/Text/]Parallelogram
A[(Text)]Cylinder
A>Text]Triangle

Declare a node once with its label, then reference it later by id alone. Wrap a label in quotes to use brackets or symbols inside it: A["count (n)"].

Edges

The operator between two nodes sets the line style and arrowheads.

SyntaxResult
A --> BArrow
A --- BLine, no arrowhead
A -.-> BDotted arrow
A ==> BThick arrow
A --o BCircle end
A --x BCross end
A <--> BArrows on both ends

Edge labels

Add a label to an edge with pipes, or inline between the dashes.

A -->|yes| B
A -- yes --> B

Chaining

Connect several nodes in one line — each pair becomes its own edge.

A --> B --> C --> D

Exporting from the canvas

The Current diagram view always reflects what is on the canvas. Use the copy button to send it to the editor. Some elements have no text equivalent — freehand drawings, images, sticky notes, plain text, and arrows that aren’t connected to two nodes — and are left out of the generated code.

Comments

Lines starting with %% are ignored.

%% a release flow
flowchart LR
build --> test --> deploy

Layout is generated automatically from the connections, so positions and colors are not stored in the code. Re-generating a diagram lays it out fresh.
