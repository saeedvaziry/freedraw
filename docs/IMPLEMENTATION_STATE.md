## Data model as-built
_(Keep in sync with code. This is the most-referenced section. Start from the model in `.plans/00-OVERVIEW.md` and record any deviations.)_

- `Element` union (`model/types.ts`): `ShapeElement | StickyElement | TextElement | FreedrawElement | ImageElement | ArrowElement`, discriminated on `type`. Matches `.plans/00-OVERVIEW.md` exactly. All `BaseElement` fields present (`id, type, x, y, width, height, rotation, style, label?`). Painters: all 11 shapes + `arrow`/`line` (Phase 5) + `text` (Phase 6) + `sticky`/`image` (Phase 8). `freedraw` modeled but not painted (no tool — pencil dropped, see deviations).
- **Sticky & image (Phase 8):** `createSticky(init)` → `StickyElement` (`type:'sticky'`, default 160×120, yellow `fill #fef08a`/`stroke #eab308`, `roundness 12`, `textAlign:'center'`); painted by `render/painters/sticky.ts` (filled `roundRect` outline + `paintLabel`). `createImage(init)` → `ImageElement` (`type:'image'`, `assetId`), sized via `fitToViewport(naturalW, naturalH, viewportW, viewportH)` = `scale = min(1, 0.8·vw/nw, 0.8·vh/nh)` (caps the longest side to 80% of the viewport, preserves aspect; a small natural is left unchanged). Painted by `render/painters/image.ts` (`drawImage` of the cached `ImageBitmap`, honoring `opacity`; neutral placeholder rect + decode-trigger when not yet cached). Both reuse the existing AABB hit-test/transform/selection/undo/styling paths — no new wiring there.
- **Text & labels (Phase 6):** two distinct concepts, per the overview. (1) **`label`** (`BaseElement.label?: { text, align, verticalAlign }`) is **baked onto** shapes/stickies/arrows — set/edited via double-click, painted by the shared `paintLabel` (shapes, centered/wrapped within bounds) and `paintArrowLabel` (arrows, at the polyline arc-length midpoint with a background plate). (2) **`text`** is a standalone `TextElement` (`createText` factory; defaults `textAlign:'left'`, 160×24) created by the Text tool. Both edited through ONE DOM `<textarea>` overlay. Committing an **empty** label removes the `label`; committing an **empty** new text element deletes it. The element being text-edited is **hidden from the scene paint** while its textarea is open (no double-render).
- **Arrows (Phase 5):** `ArrowElement` (`type: 'arrow'|'line'`) carries world-space `points[]` ([0]=start, [n-1]=end), optional `start`/`end` `Binding`s, `startArrowhead`/`endArrowhead` (`none|triangle|dot|bar`; `arrow` defaults end→triangle, `line` defaults none), and `routing` (`straight` only in V1). The arrow's `x/y/width/height` AABB is **derived from `points`** (`factory.pointsBounds`) and kept in sync on every points change (in the store recompute + tool reshape). `rotation` stays 0 for arrows — they carry baked world points, so hit-test/render skip the rotate path. `createArrow(init)` factory.
- `Style` (`model/types.ts`): as in the overview. Defaults in `model/schema.ts#defaultStyle` (`stroke #1e1e1e`, `fill transparent`, `strokeWidth 2`, `strokeStyle solid`, `opacity 1`, `roundness 0`, `fontSize 16`, `fontFamily Inter…`, `textColor #1e1e1e`, `textAlign center`).
- **`lastUsedStyle` flow (Phase 7):** `appState.lastUsedStyle` (seeded from `defaultStyle`) is the style new elements inherit. It is updated whenever the user changes style via `updateStyle`. **All create paths read it:** `ShapeTool`, `ArrowTool`, and `TextTool` pass `ctx.store.getLastUsedStyle()` as the factory `style`. (`spawnConnectedShape` and `duplicateElements` intentionally copy the SOURCE element's style instead — you want a spawned/cloned node to match its origin, not the global last-used.) Painters already honor every `Style` prop (opacity, fill, `setLineDash` per `strokeStyle`, arrowheads) — Phase 7 added NO structural painter changes; it only extracted the `solid/dashed/dotted → number[]` line-dash map into the shared `render/painters/dash.ts#dashPattern` (used by both `shape.ts` and `arrow.ts`, unit-tested).
- `Binding` (`model/types.ts`): `{ elementId, anchor: { nx, ny }, gap }`. `anchor` is normalized (0..1) within the bound shape's local bounds; `gap` (default `DEFAULT_GAP = 0`, so the arrowhead tip touches the edge) pulls the endpoint back off the edge. Bindings live on the arrow; `points[]` are derived from them.
- **UI tool state (Phase 3):** `UiState` (local-only, in `SceneStore`, NOT in the Y.Doc) gained `activeTool: ToolId` and `activeShapeType: ShapeType`. `ToolId = 'select'|'hand'|'arrow'|'text'|'sticky'|'image'|'shape'`. Defaults: `activeTool 'select'`, `activeShapeType 'rect'`. React chrome reads these via `subscribeUi`/`getUiState`; `setUiState({ activeTool, activeShapeType })` is the discrete switch action that drives both the toolbar highlight and the engine's `ToolManager` (the controller subscribes to `subscribeUi` and calls `ToolManager.setActive`).
- **Selection state (Phase 4):** `UiState.selectedIds: Set<ElementId>` and `UiState.hoveredId: ElementId|null` are now actively driven by `SelectTool` (still local-only). Selection is ephemeral editor state — NOT in the Y.Doc, so it doesn't sync/persist/undo. The controller derives the overlay chrome (selection frame, hover, marquee) from `selectedIds`/`hoveredId` each paint; React reads `selectedIds.size` for the actions bar's `hasSelection`. **Rotation is now rendered**: `Renderer` rotates each element around its center when `element.rotation` is set (was unused before Phase 4).
- Y.Doc structure: `doc.getMap('elements')` → `Y.Map<id, Y.Map<field, value>>` (each element is a flat `Y.Map`; nested objects like `style`/`label` are stored as plain JSON values inside the field, NOT nested Y types — sufficient for V1 since whole-element field updates go through `updateElement`). `doc.getArray('elementOrder')` → `Y.Array<id>` (low→high paint order). `doc.getMap('appState')` holds `camera {x,y,zoom}`, `schemaVersion`, `lastUsedStyle`. **Transaction origin** = the string `'freedraw'` (exported as `TRANSACTION_ORIGIN`); all mutations pass through `SceneStore.transact`.
- `schemaVersion`: `SCHEMA_VERSION = 1` (`model/schema.ts`), persisted on `appState`. `defaultAppState()` seeds it.

## Public contracts
_(Signatures other phases plug into. Fill as you build them.)_

- `SceneStore` API (`store/SceneStore.ts`):
  - `new SceneStore(doc?: Y.Doc)` — owns the doc, builds the plain-object mirror, attaches a single set of observers (`elements.observeDeep`, `elementOrder.observe`, `appState.observe`).
  - `getSnapshot(): SceneSnapshot` — `{ elements: Record<id, Element>, order: id[], appState }`. Returns the SAME reference until a mutation occurs (a new object is created on change) so `useSyncExternalStore` works without tearing. **The renderer/hit-test read ONLY this; never Yjs per frame.**
  - `subscribe(cb): () => void` — fires on every model mutation. Also flips `needsRender = true`.
  - `transact(fn: (api: TransactionApi) => void)` — wraps `doc.transact(fn, 'freedraw')`, then runs `resolveBoundArrows()` inside the SAME transaction (Phase 5) so bound arrows re-land in the same undo step. `TransactionApi = { addElement, updateElement, removeElement, reorder }` (camera split out in Phase 4).
  - UI channel: `getUiState(): { selectedIds: Set<id>, hoveredId: id|null, activeTool, activeShapeType }`, `setUiState(patch)`, `subscribeUi(cb)`. Local-only, NOT in the Y.Doc — won't sync/persist/undo.
  - `arrowsForShape(id): ReadonlySet<id>` — the derived `shapeId → Set<arrowId>` index, rebuilt on every element change (empty until arrows exist).
  - `needsRender: boolean` — render dirty flag; the controller wires `subscribe` → `loop.markDirty`.
  - **Undo/redo (Phase 4):** `undo()`, `redo()`, `canUndo: boolean`, `canRedo: boolean`, `stopCapturing()`, `subscribeHistory(cb)`. Backed by a `Y.UndoManager` over `[yElements, yOrder, yAppState]` with `trackedOrigins: {TRANSACTION_ORIGIN}` and `captureTimeout: 500ms`. Each gesture is one undo step: tools call `stopCapturing()` on pointerdown (break from the prior gesture) and pointerup; intra-gesture per-frame transacts merge via captureTimeout. `subscribeHistory` fires on `stack-item-added`/`stack-item-popped` for the actions bar.
  - **Camera is NOT undoable:** `transact()` no longer exposes `setCamera`. Camera now commits via a separate method `commitCamera(camera: CameraState)` under a distinct origin `CAMERA_ORIGIN = 'freedraw:camera'`, which the UndoManager does not track. (`HandTool`/`EditorController` call `store.commitCamera` instead of `api.setCamera`.)
  - **Delete/duplicate path (Phase 4, extended Phase 5):** `deleteElements(ids)` now ALSO removes arrows bound to any deleted shape (`arrowsForShape`) — the cascade-delete rule is REMOVE — all in ONE transact, pruning everything removed from `selectedIds`. `duplicateElements(ids): ElementId[]` clones (structuredClone) with new ids + a +16,+16 offset in ONE transact, appends to order, and selects the clones.
  - **Styling (Phase 7):** `updateStyle(ids, patch: Partial<Style>)` — patches `element.style` for every targeted element in ONE `transact` (so multi-select = one undo step), then folds the same `patch` into `appState.lastUsedStyle`. `updateArrowheads(ids, patch: Partial<Pick<ArrowElement,'startArrowhead'|'endArrowhead'>>)` — same, for the arrow-only top-level head fields (filters non-arrows out). `getLastUsedStyle(): Style` / `updateLastUsedStyle(patch)` — read/write the inherited style; **the `lastUsedStyle` write uses `CAMERA_ORIGIN`** (UndoManager-untracked) so it never adds its own undo step — the element edit in `updateStyle` is the single tracked step. `getSelectionStyle(): SelectionStyle` — the derived **selectionStyle slice**: shared style values across `uiState.selectedIds`, or the `MIXED` (`'__mixed__'`) sentinel per field where they disagree; falls back to `lastUsedStyle` when nothing is selected. Returns a **stable reference** until the derived values change (cached + shallow-compared) so `useSyncExternalStore` doesn't tear. `subscribeStyle(cb)` — combined `subscribe`+`subscribeUi` channel (fires on model OR selection change) for the panel. `deriveSelectionStyle(elements, fallback)` + `MIXED` are exported pure helpers (`store/selectionStyle.ts`).
  - `destroy()` — detaches observers + UndoManager.
- **`Tool` interface (`tools/Tool.ts`, Phase 3):** each tool has `readonly id` plus optional `onActivate/onDeactivate(ctx)` and `onPointerDown/onPointerMove/onPointerUp(info, ctx)` / `onKeyDown(event, ctx)`. Pointer handlers return `ToolResult | void` = `{ scene?: boolean; overlay?: boolean }` (dirty flags; the controller marks the render loop dirty if either is set). `PointerInfo = { world, screen, shiftKey, altKey, ctrlKey, metaKey, button }` (coords are pre-converted; `world` via `Camera.screenToWorld`). `ToolContext = { store, camera, setPreview(element|null), setMarquee(rect|null), setGuides(SnapGuide[]), setPortTarget(id|null) }` — `setPreview` stashes the in-progress element; `setMarquee` (Phase 4) the selection rect; `setGuides`/`setPortTarget` (Phase 5) the live snap guides and the candidate bind-target shape id (highlighted on the overlay).
  - **`ToolManager` (`tools/ToolManager.ts`):** holds `SelectTool`, `ShapeTool`, `HandTool`, `ArrowTool` (one for `arrow`, one for `line`), and a `NoopTool` per remaining inert tool id (`text`/`sticky`/`image`). Default active tool is `SelectTool`. `setActive` commits `this.active` BEFORE running deactivate/activate hooks so a hook that re-enters `setActive` (e.g. a tool clearing `hoveredId` on deactivate) short-circuits instead of recursing (Phase 4 fix). **`line` is wired in the engine but has no toolbar button yet** (only `arrow` is surfaced; see deviations).
  - **`SelectTool` (`tools/SelectTool.ts`, Phase 4):** the default tool. pointerdown decides a drag mode: handle-hit → `resize`/`rotate`; element-hit → set selection (click replaces, shift-click toggles) + `move`; empty → `marquee` (clears selection unless shift-additive). pointermove with no active drag tracks `hoveredId`. Move/resize/rotate each accumulate per-frame `updateElement` transacts; `stopCapturing()` on pointerdown+pointerup makes each gesture ONE undo step. Resize/rotate capture the element snapshot at pointerdown and transform from those originals each frame (idempotent). Single selection = rotated frame following `element.rotation`; multi-selection = axis-aligned AABB frame (rotation 0). Marquee uses `marqueeHits` (AABB-intersect, not full-containment). **Phase 5 added modes (checked first in pointerdown):** `reshape` (drag a selected arrow's endpoint square → **collapses the arrow to a clean 2-point `[start,end]` and re-routes from scratch each frame** (no stale corners), binding/unbinding by snapping onto/off a shape; drag the midpoint pill → insert/move a waypoint), `portDrag` (pointerdown on a hovered/selected shape's port → create a new arrow bound at that port and drag its free end, binding on drop). `onKeyDown` handles **Opt/Alt+Arrow** when exactly one shape is selected → `spawnConnectedShape`. Selection chrome for arrows = endpoint + midpoint handles (not the resize box).
  - **`ArrowTool` (`tools/ArrowTool.ts`, Phase 5):** one class, constructed per kind (`'arrow'`/`'line'`). pointerdown snaps the start (binds `start` if it lands on a shape); move shows a live preview + snap guides + target highlight; up binds `end` if dropped on a shape, commits if length ≥ 6, and selects the new arrow. Snap threshold = `SNAP_DISTANCE(8)/zoom` (screen-constant).
  - **`ShapeTool` (`tools/ShapeTool.ts`):** `new ShapeTool(shapeType)`, `setShapeType(type)`. `onPointerDown` records the world start; `onPointerMove` builds a preview element from `dragBounds` and calls `setPreview` (overlay dirty); `onPointerUp` clears the preview and commits via `store.transact(addElement)` if both dims ≥ `MIN_SIZE` (8). `onDeactivate` clears any preview. Exported pure helper `dragBounds(start, current, square=false): Rect` normalizes a drag to a top-left rect (Shift constrains to a square).
  - **`HandTool` (`tools/HandTool.ts`):** pans via `camera.panByScreen` on drag; commits the camera to `appState` on pointerup.
  - **`InputManager` (`input/InputManager.ts`):** `new InputManager(overlay, camera, handlers)` where `handlers = { getActiveTool, context, onResult(result), onWheel(event) }`. `attach()` wires `pointerdown/move/up/cancel` + `dblclick` + `wheel` (passive:false) on the overlay and `keydown` on `window`, builds `PointerInfo` (screen = clientXY − overlay rect; world via camera), uses pointer capture during drags, forwards to the active tool, and routes results to `onResult`. `dblclick` → `Tool.onDoubleClick` (Phase 6: begin-edit). Returns a cleanup. Wheel is delegated to the controller (camera concern), not the tool.
  - **`TextTool` (`tools/TextTool.ts`, Phase 6):** `onPointerDown` creates a `text` element at the cursor, selects it, switches `activeTool` to `select`, and calls `ctx.beginEdit` to open the editor immediately.
  - **`StickyTool` (`tools/StickyTool.ts`, Phase 8):** mirrors `TextTool` exactly. `onPointerDown` (left button) creates a sticky centered on `info.world` with `getLastUsedStyle()` **but forces the yellow sticky `fill`/`stroke`/`roundness` + `textAlign:'center'` on top** (last-used colors never erase the sticky's identity), `addElement` in one `transact` (bracketed by `stopCapturing`), selects it, switches `activeTool→select`, and `beginEdit({ target: 'label', ... })` so the label is immediately editable (the Phase-6 DOM textarea overlay + `withoutText` handle the rest with zero new code).
  - **`ImageCache` (`assets/imageCache.ts`, Phase 8):** module-class, framework/DOM-chrome-free. `new ImageCache({ loadBlob, decode?, onReady })` — `loadBlob(assetId)→Promise<Blob|undefined>` and `onReady()` (repaint) are **injected** (engine never imports persistence/React); `decode` defaults to `createImageBitmap`. `getBitmap(assetId)` returns the cached `ImageBitmap` or `undefined` and kicks off a decode; a per-id in-flight `Map` **dedupes concurrent decodes**; on success it caches and fires `onReady()` once. `set(assetId, bitmap)` seeds a just-decoded bitmap so a freshly-inserted image renders on the very next frame without a re-decode. The `image` painter reads a module-level cache set via `setImageCache(cache|null)` (`render/painters/image.ts`); the `EditorController` owns the single `ImageCache`, sets it on construct and `setImageCache(null)` on unmount.
  - **`validateImageInput(blob, maxBytes?)` (`assets/imageInput.ts`, Phase 8):** pure validator (unit-tested). Rejects non-`image/*` MIME (`reason:'mime'`) and blobs over `MAX_IMAGE_BYTES = 10 MB` (`reason:'size'`) **before** any decode/store. Returns `{ ok: true }` otherwise.
  - **`ToolManager` (Phase 8):** `INERT_TOOLS`/`NoopTool` removed entirely. `sticky` resolves to the real `StickyTool`; `image` and any unknown id fall back to `selectTool` (image never activates as a canvas tool — see UX decision below). `EditorController` gained `requestRepaint()`, `setImageBlobLoader(loader)`, `cacheImageBitmap(assetId, bitmap)`, `viewportSize`, and `screenToWorld(point)` for the app image-insert flow.
  - **`SelectTool.onDoubleClick` (Phase 6):** hit-tests the cursor; arrow → mid-arrow label edit, `text` element → body edit, other shapes → centered label edit. All route through `ctx.beginEdit`.
  - **`EditorController` edit API (Phase 6):** `subscribeEdit(listener)`, `activeEdit`, `worldToScreen(point)`, `zoom`, `measureTextSize(text, style)`, `elementCenterScreen(id)`, `resizeTextWhileEditing(id, text)` (live re-measure + re-center of a standalone text element on each keystroke), `commitText(id, target, text)`, `cancelEdit()`. The controller owns the `EditRequest` lifecycle and hides the edited element's text/label from the scene paint (`withoutText` in `Renderer.renderScene`'s `editingId` path) so the textarea never overlaps the committed render.
- **Text engine (`text/`, Phase 6):** pure, framework-free. `measure.ts`: `MeasureContext { measureWidth(text) }`, `canvasMeasureContext(ctx, fontSize, fontFamily)` (sets `ctx.font`, measures via `ctx.measureText`), `fontString`, `approximateMeasureContext` (node/test fallback), `offscreenMeasureContext` (shared lazily-created offscreen canvas; falls back to approximate when `document` is absent). `layout.ts`: `layoutText(input, measure): TextLayout { lines, width, height, lineHeight }` + `wrapText` (word-wrap within width, preserves `\n`, breaks over-long words, no-op when `width<=0`), `lineHeightFor` (1.25 ratio). `cache.ts`: `LayoutCache` keyed by `layoutKey = text|round(width)|fontSize|fontFamily|lineHeight` → returns the SAME layout object until content/style/width changes (`invalidate(id)`/`clear()`). `size.ts`: `measureTextBox(text, style): {width,height}` (content size + `TEXT_BOX_PADDING_X/Y`, floored at `MIN_TEXT_WIDTH`) — the single source for auto-sizing standalone text. `arrowLabel.ts`: `polylineMidpoint(points)` (arc-length midpoint). All unit-tested (`layout.test.ts`, `cache.test.ts`, `size.test.ts`, `arrowLabel.test.ts`, `model/text.test.ts`).
- **Text painter (`render/painters/text.ts`, Phase 6):** owns a module-level `LayoutCache`. `paintText` (standalone text element, top-aligned), `paintLabel(element)` (baked label, padded within bounds, honors `label.align`/`verticalAlign`), `paintArrowLabel(element, midpoint)` (measures unwrapped, draws a `#fafafa` plate then centered text). `paintShape` calls `paintLabel` after the stroke (so a rotated shape's label rotates with it, inside the same rotated ctx); `paintArrow` calls `paintArrowLabel(polylineMidpoint(points))`. Text is measured with the live canvas ctx → crisp at any zoom (camera transform already applied to the scene ctx).
- **Edit overlay contract (`text/edit.ts` + `ToolContext.beginEdit`, Phase 6):** `EditRequest = { elementId, target: 'text'|'label', text, world {x,y,width,height}, style, align, verticalAlign }`. A tool calls `ctx.beginEdit(request)`; the `EditorController` stores it, `stopCapturing()`s (so the edit is its own undo step), repaints (hiding the edited text element), and notifies `subscribeEdit(listener)` subscribers. The React `TextEditorOverlay` (`apps/web`) renders a positioned `<textarea>` from `controller.worldToScreen` × `controller.zoom`, matching font/size/color/align; commits on **blur** and **Enter** (Shift+Enter = newline), cancels on **Escape**. Commit calls `controller.commitText(id, target, text)` → one `transact` writing `text`/`label` (empty label→remove; empty new text→delete element). `controller.cancelEdit()` deletes a still-empty freshly-created text element. **Double-click** is routed through `InputManager` (`dblclick` → `Tool.onDoubleClick`); `SelectTool.onDoubleClick` opens the right editor for shapes (label), text (body), or arrows (mid-arrow label).
- Painter registry signature (`render/painters/index.ts`): `Painter = (ctx: CanvasRenderingContext2D, element: Element) => void`; `getPainter(type) => Painter | undefined`. Painters draw in **world space** (the scene ctx already has the camera+DPR transform applied). **As-built (Phase 3): registry is data-driven** — all 11 shape types map to a single shared `paintShape` (`render/painters/shape.ts`) that traces the element's `getOutline(...)` and applies fill/stroke (incl. `strokeStyle` dash patterns). Adding a shape = add an outline entry in `geometry/shapeOutline.ts` + add the type to the `shapeTypes` array in the registry. The Phase-2 standalone `rect.ts` painter was removed in favor of this shared painter. **Phase 5:** `arrow`/`line` map to `paintArrow` (`render/painters/arrow.ts`) — polyline + per-end arrowheads (`triangle` filled, `dot`, `bar`), arrowhead size scales mildly with `strokeWidth`.
- **`intersectRay(element, anchor, toward): Point` (`connectors/intersect.ts`, Phase 5):** crossing of segment `[toward → anchor]` with the element outline nearest `toward`. Rotation-aware; ellipse closed-form; polygons segment-vs-edge; path shapes approximate via AABB. **Note:** binding resolution no longer calls this (it pins to the anchor + minimal-bend, see below); retained as a general utility (and unit-tested for rect 3 dirs + anchor-pinning, ellipse, diamond, rotation).
- `Camera` API (`geometry/Camera.ts`): `new Camera({x,y,zoom})`; `worldToScreen/screenToWorld`; `panByScreen(dx,dy)`; `zoomToScreenPoint(nextZoom, screenPoint)` (zoom-to-cursor, keeps the world point under the cursor fixed); `viewportWorldRect(w,h)` (for culling); `state`/`setState`. Zoom clamped to `[MIN_ZOOM 0.1, MAX_ZOOM 8]` via `clampZoom`. **Camera moves are persisted to `appState` via `transact(setCamera)` on gesture end — decision: camera IS written to the doc (so it persists in Phase 9) but is a candidate to exclude from the UndoManager scope in Phase 4 so pan/zoom isn't undoable.**
- **Document persistence API (`@freedraw/persistence`, Phase 9):** `createDocumentPersistence(doc?, name?): DocumentPersistence` wraps `y-indexeddb`'s `IndexeddbPersistence`. `DocumentPersistence = { doc, provider, whenSynced: Promise<void>, clear(): Promise<void>, destroy(): void }`. **Two distinct IndexedDB databases:** `freedraw-doc` (`DOCUMENT_DB_NAME`, the Y.Doc autosave/load, owned by y-indexeddb) and `freedraw-assets` (image Blobs, Phase 8 — separate store). Autosave is automatic on every Yjs transaction; `whenSynced` resolves once the persisted state is loaded into the doc. The wiring is thin (no headless test of `IndexeddbPersistence` itself; the migration runner + validator + fit are unit-tested as pure functions instead, per the plan).
- **Schema migration + validation API (`@freedraw/engine`, Phase 9, pure):**
  - `model/migrations.ts`: `migrations: Migration[]` is an **ordered list keyed by version** — `migrations[v]` upgrades a doc FROM version `v` to `v+1`. `migrateDoc(doc)` reads `appState.schemaVersion` (missing → 0) and runs `migrations[v]` for `v` in `[current, SCHEMA_VERSION)` sequentially, then bumps `schemaVersion` to `SCHEMA_VERSION` if not already there. **V1 ships with an empty `migrations` list** (only `SCHEMA_VERSION = 1` exists) — the runner is real/ordered/unit-tested (with a fixture that injects prior-version steps). `seedAppState(doc)` fills only missing `appState` fields (schemaVersion/camera/lastUsedStyle) from `defaultAppState()`. **To add a migration:** bump `SCHEMA_VERSION` in `model/schema.ts` to N, then set `migrations[N-1] = (doc) => { … }` (transform the doc's `elements`/`order`/`appState` in place). `readSchemaVersion(doc)` is exported for boot/tests.
  - `model/serialize.ts`: `isValidScene(doc): boolean` treats the persisted doc as **untrusted** — verifies `elements` is a `Y.Map` of `Y.Map`s each shaped like an `Element` (id/type/x/y/width/height/style present + typed) and `elementOrder` is a string array. `serializeScene(snapshot): SerializedScene` and `applyScene(doc, scene)` round-trip a scene through a fresh doc (used by tests; the live app never re-serializes — y-indexeddb persists the doc directly). An empty fresh doc is valid.
- **Async boot (`apps/web/src/board/createBoard.ts`, Phase 9):** `createBoard(): Promise<Board>` is now **async**. Sequence: `createDocumentPersistence()` → `await whenSynced` → `isValidScene(doc)`? if invalid: log + `backupCorruptDoc` (base64 Y update under `localStorage['freedraw:corrupt-backup']`) + `clear()` the DB + `destroy()` + **build a fresh doc/persistence** → `migrateDoc(doc)` (wrapped in try/catch — a throw also falls back to fresh) → `seedAppState(doc)` → `new SceneStore(doc)`. The SceneStore constructor builds the mirror + rebuilds the binding index for free, so bound arrows resolve post-hydrate. Returns `{ store, persistence }`. **Corrupt/incompatible data never crashes the app — it falls back to a clean empty board.** `BoardRoute` holds `store` in state (null until hydrated) and renders a blank `BoardLoading` gate until the promise resolves, THEN mounts `CanvasHost`/`EditorController` (so the render loop starts only after sync). The `Camera` reads `store.getSnapshot().appState.camera` in its constructor → **camera restores automatically** post-hydrate.
- **Asset rehydrate (Phase 9):** unchanged from Phase 8 — lazy. After a refresh, the `image` painter's `getBitmap(assetId)` misses, the injected `setImageBlobLoader(assetRepo.getAsset)` re-loads the Blob from `freedraw-assets`, decodes to `ImageBitmap`, caches, and repaints via `onReady`. No eager pre-decode added (the lazy path was verified not to flicker noticeably).
- **`EditorController` zoom API (Phase 9):** `zoomToFit()` computes `contentBounds(snapshot)` (union of element AABBs over `order`; null when empty) and `fitCamera(bounds, vw, vh, FIT_PADDING=64)` (centers content, `clampZoom`s the fit scale), sets the camera + commits. `zoomToActualSize()` zooms to 100% around the viewport center. - **Persistence repo API (`@freedraw/persistence`, Phase 8):** hand-rolled IndexedDB (no runtime deps, postinstall-free under `--ignore-scripts`). One database **`freedraw-assets`** (v1) with a single `assets` object store keyed by `assetId` → `Blob`. **The `y-indexeddb` doc DB is a SEPARATE database `freedraw-doc`** — they are deliberately distinct. `openDatabase` is self-healing: if the `assets` store is ever missing on an existing DB it bumps the version and creates it. `createAssetRepo(factory?: IndexedDbFactory): AssetRepo` and a default `assetRepo` singleton. `AssetRepo = { putAsset(id, blob): Promise<void>; getAsset(id): Promise<Blob | undefined>; deleteAsset(id): Promise<void> }`. `db.ts` exposes `openDatabase(factory?)`, `runRequest(request)`, `ASSET_STORE`, and the `IndexedDbFactory` injection seam (`{ open(name, version): IDBOpenDBRequest }`) so tests pass an in-memory stub (`testing/memoryIndexedDb.ts`) and never need a real browser/`fake-indexeddb`. Each op opens, transacts, then `db.close()`. (Yjs `y-indexeddb` doc persistence still arrives in Phase 9; assets are a separate store and already persist.)
- **Binding & resolution API (`connectors/`, Phase 5):**
  - `binding.ts`: `anchorPoint(element, anchor): Point` (normalized anchor → world, rotation-aware), `anchorFromPoint(element, world): {nx,ny}` (world → normalized, clamped 0..1), `createBinding(element, world, gap?)`, `DEFAULT_GAP = 0` (arrowhead touches the edge).
  - `binding.anchorNormal(element, anchor)`: the outward unit normal at an anchor — **always axis-aligned** (picks the nearer edge axis; a corner resolves to one edge, never a diagonal), rotation-aware. Used for gap pull-back and orthogonal exit/entry directions.
  - `resolve.ts`: `resolveArrowPoints(arrow, elements): Point[]` — **both bound endpoints stay pinned to their fixed anchor ports** (`anchorPoint` pulled back along `anchorNormal` by `gap`), tracking the port as the shape moves/resizes/rotates. The arrow is then **orthogonally routed** (draw.io elbow style, matching `idea-examples/5.png`): each bound end pushes out a `STUB=20` stub along its edge normal, then `route()` joins the stubs with axis-aligned segments. The router is **heading-aware** — it checks whether the other end is in front of or behind each port's heading and pivots accordingly, so a port that faces *away* from the target turns perpendicular and routes around instead of doubling back through its own shape. Each end **leaves/enters its port perpendicular to the edge**; every segment is horizontal or vertical. `simplify()` dedupes coincident points and drops collinear corners, so a clear straight shot collapses to 2 points. **Fully-bound arrows are always re-derived** (they don't accumulate corners). A partially-bound/free arrow with user waypoints (`points.length > 2`) keeps its interior points; only the bound endpoint(s) re-pin. Obstacle avoidance (routing fully *around* overlapping shapes) is not implemented — the route connects the chosen ports directly. `intersectRay` is no longer used by resolution (kept as a utility). `arrowNeedsResolve(arrow)`.
  - `spawn.ts`: `spawnConnectedShape(store, source, direction): newId` — Opt+Arrow composition (new shape offset `SPAWN_GAP=80` in direction + a both-ends-bound arrow + selects the new shape), one transact.
- **Binding index API (`store/SceneStore.ts`, Phase 5):** `arrowsForShape(id): ReadonlySet<arrowId>` (the `Map<shapeId, Set<arrowId>>`, rebuilt O(n) on every element change in the observer). **Recompute-on-move:** `transact(fn)` runs `fn`, then `resolveBoundArrows()` **inside the same transaction** — it reads the live post-mutation element states from `yElements`, re-derives every bound arrow's `points` (+ AABB), and writes back only those that changed. So moving/resizing/rotating a shape and re-landing its arrows is ONE undo step, and only changed arrows are written. **Cascade delete rule = REMOVE:** `deleteElements(ids)` expands the set with `arrowsForShape(id)` for each deleted shape and removes arrows + shapes in one transact (one undo step).

**Render contracts (`@freedraw/engine`) — Renderer updated to two canvases in Phase 2:**
- `new Renderer(scene: HTMLCanvasElement, overlay: HTMLCanvasElement, grid?: Partial<GridStyle>)` — wraps both 2D contexts. Throws if either has no context. **(Phase 2 changed the Phase-1 single-canvas signature.)**
  - `Renderer.resize()` — the ONE place DPR is handled. Sizes BOTH canvases' backing store to `cssSize × devicePixelRatio` and sets their CSS size. The camera transform (incl. DPR) is applied per-paint via `setTransform`, not here.
  - `Renderer.renderScene(snapshot, camera)` — clears to `grid.background`, applies `setTransform(dpr*zoom, 0, 0, dpr*zoom, -cam.x*dpr*zoom, -cam.y*dpr*zoom)`, paints the **world-space** dotted grid (only the visible range) then culls + paints elements in `order`. Culling: skip elements whose AABB (expanded by `strokeWidth`) doesn't intersect `camera.viewportWorldRect`.
  - `Renderer.renderOverlay(camera, overlay?: OverlayState)` — clears the transparent overlay, then paints: `preview` (world-space), `hover` (only for arrows now — shape hover shows ports instead), `selection` (`SelectionFrame` box + handles, for selected shapes), `selectedArrows` (endpoint + midpoint handles; **bound endpoints render filled-accent, free endpoints hollow-white** — connection indicator per `idea-examples/2.png`/`4.png`), `ports` (4 edge-midpoint round ports for the hovered shape), `targetHighlight` (candidate bind target during a drag), `guides` (snap guide lines/points, magenta), `marquee`. `OverlayState = { preview?, selection?, selectedArrows?: ArrowElement[], hover?, ports?: Element|null, targetHighlight?: Element|null, guides?: SnapGuide[], marquee? }`. **(Phase 5 extended the Phase-4 shape.)** Overlay modules: `render/overlay/{selection,ports,arrowHandles,guides}.ts`; `ports.portAtScreen` and `arrowHandles.arrowHandleAtScreen` are the screen-space hit tests the SelectTool uses. The controller builds the `OverlayState` each paint, routing selected arrows to `selectedArrows` and selected shapes to `selection`. Scene rendering rotates each element around its center when `element.rotation` is set (arrows have rotation 0).
  - `viewportWidth/viewportHeight` getters expose CSS size.
  - `GridStyle = { spacing, dotRadius, color, background }`; defaults: spacing 24, dotRadius 1, subtle grey dots on `#fafafa`.
- `createRenderLoop(onRender: () => void): RenderLoopHandle` — unchanged flag-gated rAF loop. `{ start, stop, markDirty }`. Idle = zero paints (unit-tested + runtime-verified: 0 `fillRect` calls over 1.5s idle in Phase 2).
- `new EditorController(store, sceneCanvas, overlayCanvas)` + `.mount(): () => void` — wires store↔renderer↔camera↔loop↔tools↔input. Subscribes `store` → `loop.markDirty` and `store.subscribeUi` → `ToolManager.setActive` (so toolbar tool/shape selection drives the active tool) and sets the overlay cursor per tool (`hand`→grab, `shape`→crosshair, else default). Owns a `ToolContext` whose `setPreview`/`setMarquee` update the controller's overlay ephemerals painted each frame; `subscribeUi` now also marks the loop dirty so selection/hover changes repaint the overlay. Delegates overlay pointer/key/wheel to an `InputManager`; wheel zoom/pan and Space/middle-drag panning stay controller-owned. **Space/middle-drag pan uses capture-phase listeners with `stopImmediatePropagation` so it overrides any active tool** (e.g. you can pan while the shape tool is active). Camera committed to `appState` on gesture end (drag pointerup; wheel/pinch debounced 150ms). Returns the unmount cleanup.

**Geometry contracts (`@freedraw/engine`) — Phase 3:**
- `geometry/shapeOutline.ts` is the SINGLE source of shape geometry, shared by the painter (and later hit-test + connector intersection). `getOutline(type, bounds): Outline | undefined`. `Outline` is a union: `{ kind: 'polygon', points }` (rect, diamond, triangle, hexagon, parallelogram, star), `{ kind: 'ellipse', cx, cy, rx, ry }`, or `{ kind: 'path', build(ctx|Path2D) }` (roundRect, cylinder, cloud, heart — curved). `traceOutline(ctx|Path2D, outline)` appends the outline to a path (caller does beginPath/fill/stroke). `pointInPolygon(point, polygon)` is the ray-cast test, reused by Phase-4 narrow-phase hit-testing.

**Hit-test & transform contracts (`@freedraw/engine`) — Phase 4:**
- **`geometry/hitTest.ts` — the tiered hit-test contract.** All world-space; convert the pointer once. `hitTest(point, snapshot): Element | null` iterates `order` front-to-back, broad-phase rejects via the element's rotation-aware AABB (`rotatedAabb`, expanded by `strokeWidth/2 + 6px tolerance`), then narrow-phase per type: arrows/lines/freedraw = distance-to-polyline ≤ `strokeWidth/2 + tol`; text/sticky/image = AABB; shapes = `hitTestElement` reusing `getOutline` (ellipse = normalized radius; polygon = `pointInPolygon`; unfilled shapes also hit a stroke band near the outline). Rotation handled by un-rotating the pointer into element-local space (`toLocalPoint`) before the per-type test (arrows carry baked world `points`, so they skip the un-rotate). `HIT_TOLERANCE = 6`. Helpers: `hitTestElement(point, element)`, `elementBounds`, `elementCenter`, `selectionBounds(elements): Rect|null` (union of rotation-aware AABBs), `marqueeHits(rect, snapshot): Element[]` (AABB-intersect).
- **`geometry/selectionFrame.ts`:** `selectionFrameFor(elements): SelectionFrame|null` — single element → `{ bounds: AABB, rotation: element.rotation, center }`; multiple → axis-aligned union AABB with `rotation 0`.
- **`geometry/handles.ts`:** `SelectionFrame = { bounds: Rect, rotation, center }`. 8 resize handles (`nw n ne e se s sw w`) + a `rotate` handle. `resizeHandlesScreen(frame, camera)` / `rotateHandleScreen(frame, camera)` give screen-space positions (handles drawn screen-space → constant size at any zoom); `handleAtScreen(screen, frame, camera): HandleId|null` is the screen-radius hit test (handles win over elements). Rotate handle sits `ROTATE_HANDLE_OFFSET=24px` above the frame top-center, along the frame's rotation.
- **`geometry/transform.ts`:** `resizedBounds(frame, handle, pointerWorld): Rect` (un-rotates pointer into frame-local space, moves the dragged edges, keeps opposite edges fixed, min dimension 1); `resizeElements(elements, frame, nextBounds): {id, patch}[]` scales each element's x/y/w/h proportionally within the frame; `rotationFor(frame, pointer)` = `atan2(dy,dx)+π/2` (used for single-element absolute rotation; group rotate applies a delta around the frame center to captured originals).
- **`geometry/snap.ts` (Phase 5):** `snapEndpoint(world, snapshot, { threshold, origin?, ignoreId? }): { point, target, guides }` resolves an arrow endpoint in priority order: (1) nearest **edge port** within `threshold` → bind there (fixed port); (2) else if the pointer is **anywhere over a shape body** (`hitTestElement`) → bind, snapping to that shape's **nearest edge port**; (3) else free. **There is no 45°/angle snapping** (removed — it conflicted with orthogonal routing and drew a misleading dashed guide). `shapeAnchors` returns the **4 edge-midpoint ports only — the center is NOT a snap/bind target** (binding to center would land the endpoint inside the shape). Returns the resolved `point`, the `target` shape (for binding), and `SnapGuide[]` (only the `point` kind is emitted now). `SNAP_DISTANCE = 8` screen px (tools divide by zoom). Arrows/lines are never snap targets.

### Carried-over notes
- Orthogonal arrow auto-routing: **implemented for bound arrows** (Phase 5 post-review; perpendicular port exit/entry + axis-aligned segments). Obstacle avoidance is the deferred item above.
- **Performance smoke check (Phase 9):** culling (`viewportWorldRect` AABB reject) + the flag-gated rAF loop (idle = 0 paints, verified Phase 2) are intact and unchanged this phase; the render loop starts only AFTER `whenSynced`, so boot does no work until the doc is hydrated. ~300-element pan/zoom relies on the same cheap mirror-iteration path that's been in place since Phase 2 (no per-frame Yjs reads). No dirty-rect clipping needed for V1.
- **StrictMode async boot:** `BoardRoute`'s effect creates the persistence async; the effect cleanup `destroy()`s the provider (and a `cancelled` flag destroys a late-resolving first run). Two `IndexeddbPersistence` connections to `freedraw-doc` may briefly coexist on the dev double-mount — y-indexeddb tolerates this (standard pattern). The `SceneStore` itself is still never destroyed mid-use (per the Phase-2 note).
- **Binding index rebuild (Phase 2):** `SceneStore.rebuildBindingIndex()` runs a full O(n) rebuild on every element change rather than a targeted per-id update. Index is always consistent; only a perf concern at large scale and there are no arrows until Phase 5. Make incremental when it measurably matters.
- **Render loop idle (Phase 2):** the rAF loop keeps a standing frame request while running (skips paint when not dirty → 0 paints, verified). A fully on-demand one-shot-rAF model (no standing frame) would save battery further; deferred — current model already meets the 0-paint mandate.
- **Camera commit on wheel/pinch (Phase 2):** debounced to gesture-end (`WHEEL_COMMIT_DELAY` 150ms) so one Yjs transaction is written per zoom/scroll gesture, not per tick. Drag-pan already commits once on `pointerup`/`pointercancel`. As of Phase 4 camera commits use `CAMERA_ORIGIN` (untracked by the UndoManager) so they never enter undo history regardless of frequency.
- **Group resize ignores per-element rotation (Phase 4):** when resizing a multi-selection that contains rotated elements, each element's x/y/w/h is scaled within the axis-aligned frame but its own rotation is preserved (no shear). This matches basic Excalidraw group-resize behavior and is correct for the common axis-aligned case; a fully rotation-correct group resize (transform each element's local frame) is deferred until it matters.
- **Hover highlight while a tool other than select is active:** `hoveredId` is only set by `SelectTool`, so hover highlight/ports show only with the select tool active. Selection chrome, however, renders regardless of the active tool (e.g. after Cmd+A), which is intended.
- **`intersectRay` path-shape approximation (Phase 5):** roundRect/cylinder/cloud/heart bind to their AABB rectangle, not the true curved edge (see deviations). Arrows still land sensibly; precise curved intersection deferred.
- **Bound-arrow recompute is O(all arrows) per transact (Phase 5):** correct and cheap for V1; make index-targeted if arrow count grows. Bound arrows now orthogonally route (perpendicular port exit/entry, axis-aligned segments); no obstacle avoidance yet.
- **`line` tool not in the toolbar (Phase 5):** engine-ready, no UI button yet (see deviations).
- **`freedraw` element type (Post-V1):** now CREATED by `FreedrawTool` and rendered by the `freedraw` painter (perfect-freehand). No longer dead — see the 2026-06-03 deviation.
- **Image obstacle/large-asset handling (Phase 8):** images fit to 80% of the viewport on insert; no downscaling of the stored blob, no thumbnailing, no LRU eviction on the `ImageCache` (bitmaps live for the session). Fine for V1; add eviction/thumbnails if memory matters on image-heavy boards.
